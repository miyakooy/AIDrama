# TensorsLab Studio 双模产品重构设计文档 (PRD)

## 1. 概述：为什么需要双模架构？
在 AI 影视创作领域，用户群体呈现出明显的**两极分化**：
1. **轻量级用户 (C端/小白/泛娱乐)**：他们有脑洞和灵感，但缺乏专业影视知识（景别、运镜、分镜规划），需要的是“陪伴式、引导式、低门槛”的傻瓜体验。
2. **专业创作者 (B端/导演/动画师)**：他们有明确的分镜脚本，对角色的连续性、画面的构图、镜头的推拉摇移有严苛的控制欲，需要的是“确定性、高效率、批量化”的工业级控制台。

**TensorsLab Studio 解决方案**：
摒弃传统的单一工具形态，重构为一个**「Agent 陪伴向导」与「专业工业级工作台」**深度融合的双模平台。
- **Magic Creator (轻量模式)**：以角色扮演 Agent 为核心，像玩游戏一样完成项目前期构建。
- **Director Studio (专业模式)**：以表格和时间线为核心，提供原子级的参数调优。

---

## 2. Magic Creator (轻量级 Agent 模式)

### 2.1 核心设计理念
- **拟人化陪伴 (Pet Companion)**：将冷冰冰的大模型能力（剧本解析、角色抽取、场景生成）封装为三个具有人格的 AI Agent（🐱艺术总监、🦉编剧、🦊角色设计师）。
- **聊天式交互 (CUI)**：摒弃复杂的表单填写，用户只需在一开始输入一句话，后续所有的参数配置（风格、时长、情绪）均在群聊流中通过选择题完成。
- **无限画板展现 (Infinite Canvas)**：左侧聊天沟通，右侧画板实时“掉落” Agent 产出的资产卡片（剧本大纲、角色设定图）。

### 2.2 用户旅程演示 (Demo Flow)
1. **破冰唤醒**：用户在极简首页输入“TensorsLab mia和她可爱小猫的一天”。
2. **总监接单**：进入左右分栏的 Workspace。🐱艺术总监首先响应，询问用户短片的对白语言（中文/英文）和情绪风格（治愈/温柔/有趣）。
3. **编剧介入**：用户确认后，总监邀请 🦉编剧 入群。编剧生成剧本文本，右侧画板弹出《我的剧本》卡片。
4. **视觉定调**：编剧邀请 🦊角色设计师 入群。设计师提取剧本角色（Mia、小猫），在右侧画板弹出两张带图的角色档案卡片，并提供“加到资产库”按钮。
5. **平滑升维**：轻量级用户在此即可获得满足感；若需进一步渲染视频，点击右上角“进入专业工作台”，携带所有已生成的上下文无缝切入专业模式。

---

## 3. Director Studio (专业级工业模式)

### 3.1 核心设计理念
- **确定性 (Determinism)**：一切皆可手动编辑。剧本的每一句话、角色的六视图垫图、每一个镜头的 Prompt 和 Seed 值全部暴露。
- **高密度的信息展示**：采用数据网格 (Data Grid)、瀑布流和时间轴 (Timeline) 布局，提升单屏信息密度。
- **批量与并发 (Batching)**：一键对几十个镜头进行组级并发渲染，最大化利用 API 的并发限制。

### 3.2 核心模块
- **角色库/场景库**：支持多角度参考图（Character Bible），支持通过 ControlNet/IPAdapter 锁定特征。
- **分镜工作台**：列表视图，每一行是一个镜头。包含画面提示词、镜头语言（Pan/Zoom）、视频提示词、参考底图、语音配音轨道。
- **Seedance 2.0 / S-Class**：专门为长视频串联设计的组级视频生成调度中心。

---

## 4. 底层架构融合策略

这两个看似割裂的模式，在代码底层**完全共享同一套 Zustand Store 和本地文件存储系统**。

1. **统一数据模型 (Data Schema)**：
   - 无论是在 Magic Creator 的聊天卡片里生成的“角色”，还是在专业工作台里手动添加的“角色”，最终都序列化为同一个 `Character` 对象并保存在 `characters.json` 中。
2. **状态拦截与重定向 (State Routing)**：
   - 软件启动时，若检测到用户没有历史项目，默认进入 Magic Creator 首页（降低流失率）。
   - 用户在 Agent 画板点击“进入专业工作台”时，只是单纯的 `setActiveTab('director')` 视图切换，数据无需任何显式传递，因为上下文早已在 Zustand 中同步完毕。
3. **API 路由 (Feature Router)**：
   - Magic Creator 里的 🐱艺术总监 和专业版里的“一键剧本解析”底层调用的是同一个 `script_analysis` 模型配置。

## 5. 结语
这次重构让 TensorsLab Studio 不仅是一个“好用”的工具，更是一个“好玩”的平台。它用 OiiOii 风格的轻量交互捕获创意，用工业级的底层管线落地生产，真正做到了“下限低（易上手）、上限高（专业可控）”。
